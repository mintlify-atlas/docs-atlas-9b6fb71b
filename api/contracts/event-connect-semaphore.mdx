---
title: EventConnectSemaphore
description: Protocol contract for verifying Semaphore zero-knowledge proofs
---

The EventConnectSemaphore contract integrates with the Semaphore protocol to verify zero-knowledge proofs for group membership. It prevents double-signaling by tracking nullifier hashes and supports proof verification against both current and expired Merkle tree roots.

## Contract overview

```solidity
contract EventConnectSemaphore is IEventConnectSemaphore
```

This contract works in conjunction with the EventConnect contract to verify Semaphore proofs for off-chain groups.

## State variables

### verifier


### eventConnect

```solidity

```

### nullifierHashes

```solidity

```

Tracks used nullifier hashes per group to prevent double-signaling.

## Functions

### verifyProof

Verifies a Semaphore zero-knowledge proof for group membership and saves the nullifier hash to prevent double-signaling.

```solidity
function verifyProof(
    uint256 groupId,
    uint256 merkleTreeRoot,
    uint256 merkleTreeDepth,
    uint256 signal,
    uint256 nullifierHash,
    uint256 externalNullifier,
    uint256[8] calldata proof
) external
```

<ParamField path="groupId" type="uint256">
  ID of the group to verify membership against
</ParamField>

<ParamField path="merkleTreeRoot" type="uint256">
  Root of the Merkle tree used in the proof
</ParamField>

<ParamField path="merkleTreeDepth" type="uint256">
  Depth of the Merkle tree (e.g., 20 for Semaphore)
</ParamField>

<ParamField path="signal" type="uint256">
  The signal being signed (application-specific data)
</ParamField>

<ParamField path="nullifierHash" type="uint256">
  Unique hash derived from identity and external nullifier
</ParamField>

<ParamField path="externalNullifier" type="uint256">
  Application-specific nullifier to scope the proof
</ParamField>

<ParamField path="proof" type="uint256[8]">
  Zero-knowledge proof data array
</ParamField>

**Events emitted:**
- `ProofVerified(uint256 indexed groupId, uint256 indexed merkleTreeRoot, uint256 indexed externalNullifier, uint256 nullifierHash, uint256 signal)`

**Reverts with:**
- `EventConnect__MerkleTreeRootIsNotPartOfTheGroup()` - The provided Merkle root was never part of the group
- `EventConnect__MerkleTreeRootIsExpired()` - The Merkle root has expired based on the group's fingerprint duration
- `EventConnect__YouAreUsingTheSameNullifierTwice()` - The nullifier hash has already been used for this group
- Semaphore verifier errors if the proof is invalid

**Example usage:**

```solidity
// Verify a proof for event attendance
uint256 groupId = 1;
uint256 merkleTreeRoot = 0x123456...;
uint256 merkleTreeDepth = 20;
uint256 signal = uint256(keccak256("I attended this event"));
uint256 nullifierHash = 0xabcdef...;
uint256 externalNullifier = uint256(keccak256("event-123"));
uint256[8] memory proof = [...]; // Generated by Semaphore client

semaphore.verifyProof(
    groupId,
    merkleTreeRoot,
    merkleTreeDepth,
    signal,
    nullifierHash,
    externalNullifier,
    proof
);
```

## Merkle root expiration logic

The contract supports verification against old Merkle tree roots to handle cases where the group membership changes between proof generation and verification:

1. If the provided `merkleTreeRoot` matches the current group fingerprint, verification proceeds immediately
2. If it doesn't match, the contract checks:
   - Whether the root was ever part of the group (via `getFingerprintCreationDate`)
   - Whether the root is still within its validity period (based on `fingerprintDuration`)
3. Expired or unknown roots cause the transaction to revert

**Example scenario:**

```solidity
// Group updated at timestamp 1000
// Fingerprint duration: 3600 seconds (1 hour)
// User generates proof at timestamp 1500
// User submits proof at timestamp 2000

// Current time: 2500
// Old root expires at: 1000 + 3600 = 4600
// Result: Proof is still valid

// Current time: 5000
// Result: EventConnect__MerkleTreeRootIsExpired() error
```

## Errors

### EventConnect__YouAreUsingTheSameNullifierTwice

```solidity
error EventConnect__YouAreUsingTheSameNullifierTwice()
```

Thrown when attempting to verify a proof with a nullifier hash that has already been used for the specified group. This prevents double-signaling attacks.

### EventConnect__MerkleTreeRootIsExpired

```solidity
error EventConnect__MerkleTreeRootIsExpired()
```

Thrown when the Merkle tree root has exceeded its validity period as defined by the group's fingerprint duration.

### EventConnect__MerkleTreeRootIsNotPartOfTheGroup

```solidity
error EventConnect__MerkleTreeRootIsNotPartOfTheGroup()
```

Thrown when the provided Merkle tree root was never part of the specified group's history.
