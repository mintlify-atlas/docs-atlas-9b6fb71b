## Overview

Circom circuits are the foundation of your zero-knowledge proof system. This guide walks you through building, compiling, and testing Circom circuits using the EdDSA signature verification example.

## Circuit structure

The main circuit file uses the EdDSAPoseidonVerifier from circomlib to verify EdDSA signatures:

```circom
pragma circom 2.0.0;
include "../../node_modules/circomlib/circuits/eddsaposeidon.circom";

template Sample() {
    // public
    signal input M;
    signal input Ax;
    signal input Ay;
    // private
    signal input S;
    signal input R8x;
    signal input R8y;

    component verifier = EdDSAPoseidonVerifier();

    verifier.enabled <== 1;
    verifier.Ax <== Ax;
    verifier.Ay <== Ay;
    verifier.S <== S;
    verifier.R8x <== R8x;
    verifier.R8y <== R8y;
    verifier.M <== M;
}

component main { public [M, Ax, Ay] } = Sample();
```

## Building circuits

<Steps>
  <Step title="Install dependencies">
    First, ensure all dependencies are installed:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
  </Step>

  <Step title="Compile the circuit">
    The build process compiles circuits, generates zkeys, and exports verifiers:

    <CodeGroup>
    ```bash npm
    npm run build
    ```

    ```bash yarn
    yarn workspace circuits build
    ```

    ```bash pnpm
    pnpm --filter circuits build
    ```
    </CodeGroup>

    This command runs three operations:
    1. **Compile**: Converts `.circom` files to R1CS, WASM, and symbols
    2. **Export zkey**: Generates proving keys using Powers of Tau ceremony
    3. **Export verifier**: Creates Solidity verifier contracts
  </Step>

  <Step title="Run tests">
    Test your circuit to ensure it works correctly:

    <CodeGroup>
    ```bash npm
    npm run test
    ```

    ```bash yarn
    yarn workspace circuits test
    ```

    ```bash pnpm
    pnpm --filter circuits test
    ```
    </CodeGroup>
  </Step>
</Steps>

<Note>
The build process copies compiled artifacts (`main.zkey` and `main.wasm`) to the app's public directory automatically.
</Note>

## Testing circuits

Create test files in the `test/` directory. Here's an example test:

```typescript
import { ZKPClient } from "../src/client";
import { EdDSA } from "../src/eddsa";
import { BigNumber } from "ethers";
import fs from "fs";
import path from "path";

describe("Test zkp circuit and scripts", function () {
  let client: ZKPClient;
  let eddsa: EdDSA;

  beforeEach(async () => {
    const wasm = fs.readFileSync(
      path.join(__dirname, "../../circuits/zk/circuits/main_js/main.wasm")
    );
    const zkey = fs.readFileSync(
      path.join(__dirname, "../../circuits/zk/zkeys/main.zkey")
    );
    client = await new ZKPClient().init(wasm, zkey);
    eddsa = await new EdDSA("0xABCDEF12...").init();
  });

  it("Should able to prove and verify the zkp", async function () {
    const message = BigNumber.from("0xABCDEF00...");
    const signature = await eddsa.sign(message);
    
    const proof = await client.prove({
      M: message.toBigInt(),
      Ax: eddsa.scalarPubKey[0],
      Ay: eddsa.scalarPubKey[1],
      S: signature.S,
      R8x: eddsa.babyjub.F.toObject(signature.R8[0]),
      R8y: eddsa.babyjub.F.toObject(signature.R8[1]),
    });
    
    expect(proof).not.to.eq(undefined);
  });
});
```

## Build scripts

The `package.json` includes several useful scripts:

- `compile`: Compiles all `.circom` files in the `circuits/` directory
- `export:sample-zkey`: Generates proving keys from compiled R1CS files
- `export:verifier`: Exports Solidity verifier contracts
- `build`: Runs all three steps above
- `test`: Runs the test suite with Mocha

## Output files

After building, you'll find:

- `zk/circuits/main_js/main.wasm` - WASM file for proof generation
- `zk/zkeys/main.zkey` - Proving key
- `zk/verifiers/Main.sol` - Solidity verifier contract

<Warning>
The build process uses Powers of Tau ceremony file `powersOfTau28_hez_final_15.ptau`. This is suitable for development but not for production use.
</Warning>
