# Quickstart

Get your first EventConnect zkApp running quickly with this step-by-step guide. You'll compile circuits, deploy contracts, and run a working demo application.

<Note>
  This quickstart assumes you already have Node.js, Rust, and Circom installed. If not, see the [installation guide](installation) first.
</Note>

## Clone the repository

Start by cloning the EventConnect boilerplate repository:

```bash
git clone https://github.com/wanseob/zkp-app-boilerplate
cd zkp-app-boilerplate
```

## Install dependencies

EventConnect uses Yarn workspaces to manage multiple packages. Install all dependencies with a single command:

```bash
yarn
```

This installs dependencies for all three workspaces:
- `circuits` - Circuit compilation and proof generation libraries
- `contracts` - Ethereum development tools and testing frameworks  
- `app` - React frontend and Web3 integration libraries

<Info>
  EventConnect requires Node.js version 16.x. Check your version with `node --version`.
</Info>

## Build the project

The build process compiles circuits, generates verification keys, exports Solidity verifiers, and compiles smart contracts:

<Steps>
  <Step title="Run the build command">
    ```bash
    yarn build
    ```
    
    This command executes two sub-tasks:
    - `yarn build:circuits` - Compiles Circom circuits and generates proving/verification keys
    - `yarn build:contracts` - Compiles Solidity contracts and generates TypeScript types
  </Step>
  
  <Step title="Wait for circuit compilation">
    Circuit compilation takes several minutes on first run. You'll see output like:
    
    ```
    template instances: 1
    non-linear constraints: 1234
    linear constraints: 0
    public inputs: 3
    private inputs: 3
    public outputs: 0
    wires: 4567
    labels: 8901
    ```
  </Step>
  
  <Step title="Verify build artifacts">
    After successful compilation, check that these artifacts were created:
    
    ```bash
    ls circuits/zk/circuits/main_js/main.wasm  # Circuit WASM file
    ls circuits/zk/zkeys/main.zkey             # Proving key
    ls circuits/zk/verifiers/Main.sol          # Solidity verifier
    ls contracts/artifacts                     # Compiled contracts
    ```
  </Step>
</Steps>

<Warning>
  The first build generates trusted setup parameters using `powersOfTau28_hez_final_15.ptau`. For production applications, you must perform a proper trusted setup ceremony.
</Warning>

## Run the demo

EventConnect includes a complete demo that starts a local Ethereum node and runs the full zkApp workflow:

```bash
yarn demo
```

The demo performs these operations:

<Steps>
  <Step title="Start local network">
    Launches a Hardhat local Ethereum node on `localhost:8545`
  </Step>
  
  <Step title="Deploy contracts">
    Deploys the verifier contract and zkApp contract to the local network
  </Step>
  
  <Step title="Generate proofs">
    Creates three EdDSA key pairs, signs a message with each, and generates a ZK proof verifying all three signatures
  </Step>
  
  <Step title="Submit transaction">
    Sends the proof to the smart contract for on-chain verification and records the result
  </Step>
</Steps>

You should see output confirming each step:

```

```

## Run tests

Verify everything works correctly by running the test suites:

<CodeGroup>

```bash Circuit tests
yarn workspace circuits test
```

```bash Contract tests
yarn workspace contracts test
```

```bash App tests
yarn workspace app test
```

</CodeGroup>

All tests should pass:

```
Test zkp circuit and scripts
  ✓ Should able to prove and verify the zkp (2547ms)

1 passing (3s)
```

<Info>
  Circuit tests take longer because they perform proof generation. Expect 2-5 seconds per test.
</Info>

## Common issues

<AccordionGroup>
  <Accordion title="Build fails with 'circom: command not found'">
    Circom is not installed or not in your PATH. Follow the [Circom installation guide](https://docs.circom.io/getting-started/installation/) to install it.
    
    Verify installation:
    ```bash
    circom --version
    ```
  </Accordion>
  
  <Accordion title="Node version error during install">
    EventConnect requires Node.js 16.x. Use a version manager like `nvm` to switch versions:
    
    ```bash
    nvm install 16
    nvm use 16
    ```
  </Accordion>
  
  <Accordion title="Out of memory during circuit compilation">
    Large circuits require significant memory. Increase Node's memory limit:
    
    ```bash
    export NODE_OPTIONS="--max-old-space-size=8192"
    yarn build:circuits
    ```
  </Accordion>
  
  <Accordion title="Demo fails with 'port 8545 already in use'">
    Another process is using port 8545. Find and kill it:
    
    ```bash
    lsof -ti:8545 | xargs kill -9
    yarn demo
    ```
  </Accordion>
</AccordionGroup>
